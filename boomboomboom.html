Boom Boom Boom!

Boom Boom Boom is an interactive music visualisation web-app designed for use at parties featuring graphics, animations, photos and effects which play in time to music, and can be controlled by the user.

This is a personal project I worked on to did to dig deeper into the Web Audio API, Two.js, Web-Sockets, and Node.js. Have a go.

The best way to find out about it is to play with it..

Audio Analysis with the Web Audio API

I started the project by experimenting with the Web Audio API in order to access and analyse ambient music through the computer's microphone.

Once I had wrapped my head around the API and how it works, I experimened with how to analyse the audio to find out which information I could get from the music to use to control my visuals.

Using the FFT data from the analysis, I gathered information including times of peaks, average frequency, times of high / low frequency variation, average amplitude, BPM and also events for peaks (hard / soft / high frequency / low frequency), and drops / jumps in frequency, amplitude and BPM.

Interactive Visuals

In order for the app to be interesting, I wanted the user to be able to choose from a variety of visuals, and the final app includes graphics, text, animations, photos, illustrations and effects.

I started with creating the basic graphic elements for the App, which are created using Two.js.These consist of four different circles of different sizes and positions, for hard, soft, high and low beats. The animated strips are also implimented using Two, by animating the progression of the stroke.

I then moved onto the colours, which in automatic mode are controlled by the frequency of the music. I made a colour scheme which all the colours are picked from, and then using the HSV colour model, the colours are adjusted to suit the music or the users choice.

The other graphics and effects are implimented and controlled using plain CSS3 and Javascript.

Mobile Device Controller using Web Sockets

One of the features of the App is that users can connect their mobile device to use as a custom-controller.

In order to impliment this, I built the app on a Node Express server which re-directs the user to either the mobile or desktop version of the site depending on the device. On the desktop site, a code is generated which the user must input on the mobile site.

When the user presses a key on the mobile device controller, this sends an event to the server using Web Sockets (implimented with Socket.io), which the desktop client listens to, and only reacts when the code matches the one it previously generated.
